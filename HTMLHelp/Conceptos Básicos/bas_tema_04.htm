Plan de Implementación

Para llevar a cabo la Implementación de DOBRA 7.0 es necesario cumplir las siguientes actividades generales:

1.- Instalación del Sistema
2.- Establecer Parámetros Básicos
3.- Ingresar Catálogos de Datos
4.- Establecer Opciones de Módulos
5.- Ingresar Saldos Iniciales
6.- Crear Asiento de Diario Inicial

Instalación del Sistema

Esta actividad comprende lo siguiente:

Actividad

Explicación
Instalación del Servidor de Base de Datos Instalar la versión de SQL Server de propiedad del CLIENTE en el Servidor.
Instalar Base de Datos Dobra Instalar una versión de la Base de Datos de DOBRA vacía.
Instalación de Licencias de Dobra Instalar el Producto de Software DOBRA en cada computadora de los usuarios.
Activación de Licencias de Dobra Activar el Producto de Software.
Crear Conexiones ODBC Crear las respectivas conexiones desde los usuarios hasta la Base de Datos.

Establecer Parámetros Básicos

Después de instalado el Sistema es necesario establecer los siguientes parámetros básicos:

Parámetros

 Explicación
Sucursal Predeterminada Si no existen, debe crearse por lo menos la sucursal MATRIZ en Herramientas, Sistema, Preferencias Locales.
Divisa Predeterminada Si no existen, es necesario crear por lo menos una divisa Base, en este caso la divisa DÓLARES. Si el sistema solicita ejecutar un proceso de cambio de divisa, acepte los cuadros de díalogo hasta que termine.  Esta opción se encuentra en Herramientas, Sistema, Divisas.
Datos de la Empresa Es necesario establecer los datos de la empresa y principalmente el nombre del archivo de LOGOTIPO. Esto es necesario para la impresión de varios reportes del sistema. Sólo debe escribir el nombre del archivo, NO la ruta completa. Estos parámetros se establecen en Herramientas, Sistema, Empresa.
Rutas de Archivos Es necesario establecer las rutas o directorios en donde se encuentran los archivos de formatos preimpresos de Facturas o Comprobantes; y además el directorio en donde se puede localizar el archivo de Logotipo. Estas rutas se establecen para cada computadora; es decir, que en una computadora la ruta de reportes puede ser F: y en otra puede ser G:. De igual forma el logotipo puede estar copiado en la unidad C: de cada máquina, entonces la ruta sería: C:\DOBRA7 por ejemplo. Estas opciones se establecen en Herramientas, Sistema, Preferencias Locales. Debe reiniciar DOBRA para que los cambios tengan efecto.
División Predeterminada Si no existen, es necesario crear al menos una División o Línea de Negocio. Esto se puede realizar en Herramientas, Sistema, Divisiones.
Ventana de Inicio Si no existen, es necesario crear al menos una División o Línea de Negocio. Esto se puede realizar en Herramientas, Sistema, Divisiones.

NOTA: Reinicie DOBRA para que los cambios tengan efecto.

Esta clasificación se establece en base al conocimiento de las Etapas del Flujo de Dinero que reconoce DOBRA®. Mas información ir a Herramientas, Caja y Bancos, Cuentas.

Establecer opciones de Módulos

Las opciones en cada módulo son parámetros especiales que indican al sistema que Cuentas Contables usar para generar los asientos automáticos, los archivos de formatos preimpresos, entre otros. Las opciones se encuentran en la sección de mantenimiento de cada módulo. Se deben establecer las opciones en los siguientes módulos:

Módulo Explicación
Contabilidad Las Opciones de contablidad sirve para Determinar hasta qué fecha quedará bloqueado el ingreso de transacciones en el sistema; es decir, los usuarios no podrán guardar transacciones con fechas anteriores a la de este parámetro. Esta opción se encuentra en   Herramientas, Contabilidad, Opciones.
Caja y Bancos Las opciones sirve para elejir la caja predeterminada para la máquina.   Esta opción se encuentra en Herramientas, Caja y Bancos, Opciones.
Empleados Las opciones sirve para establecer parámetros iniciales para trabajar con el Módulo de Empleados. Esta opción se encuentra en Herramientas, Empleados, Opciones.
Activos Fijos Las opciones de activo fijo contiene el cuadro Cta. Descuento en Compra que sirve para acreditar un descuento en compra a una de las cuentas del fichero de plan de cuentas, el descuento de compra.  Esta opción se encuentra en Herramientas, Activos Fijos, Opciones.
Proveeduría las opciones de proveeduría tiene dos secciones acreedores y proveeduría.  Esta opción se encuentra en Herramientas, Proveeduría, Opciones
Acreedores Las opciones de acreedores sirve para seleccionar el rubro perteneciente a los débitos a favor.  Esta opción se encuentra en Herramientas, Acreedores, Opciones.
Clientes Las opciones de clientes permite relacionar cuentas contables o rubros a las opciones presentadas en éste fichero que la empresa utilice para trabajar con sus clientes. Esta opción se encuentra en Herramientas, Clientes, Opciones.
Compras Las opciones sirve para registrar las cuentas Contables que serán afectadas por las opciones del módulo de Compras.   Esta opción se encuentra en Herramientas, Compras, Opciones.
Importaciones Las opciones de importaciones permite relacionar cuentas contables o rubros a las opciones presentadas en éste fichero que la empresa utilice para trabajar con sus clientes. Esta opción se encuentra en Herramientas, Importaciones, Opciones.
Ventas Las opciones sirve para registrar las cuentas Contables que serán afectadas por las opciones del módulo de Ventas. Además se ingresar los Formatos de Impresión para Facturas y Letras de Cambio.  Herramientas, Ventas, Opciones.

Ingresar Saldos Iniciales

El ingreso de saldos iniciales es conocido como un Proceso, permite establecer los valores iniciales de deudas de clientes, saldos de inventarios, etc. pero NO genera asientos contables. Luego de ingresar los saldos iniciales de un respectivo Módulo, se pueden imprimir reportes de saldos para usar es información en el ingreso del Balance Inicial. Los saldos iniciales que deben ingresarse son:

Módulo

Explicación
Caja y Bancos Ingrese los saldos reales desde sus cuentas de efectivo y sus cuentas bancarias. Las cuentas de clase CAJA deben iniciar con saldo cero. En el caso de cheques girados y no cobrados, se recomienda ingresar el total de cheques en una cuenta por pagar (PASIVO) en el balance inicial; luego , al iniciar el ingreso de transacciones, se deben girar los cheques desde el módulo de Caja y Bancos mediante Comprobantes de Ingreso, cruzando con la cuenta por pagar utilizada en el Balance Inicial. Para más información ir a Herramientas, Caja y Bancos, Saldos Iniciales.
Empleados Se deben ingresar por cada empleado, cada una de las deudas por préstamos u otros conceptos junto con sus fechas de vencimiento. Estas deudas aparecerán en el futuro al momento de crear Roles de Pago. Para más información ir a Herramientas, Empleados, Saldos iniciales.
Activos Fijos Los activos fijos no tienen proceso de Saldo Inicial. Los activos son entidades únicas que tienen un valor de compra, y su saldo inicial está dado por el valor con que se ingresó al sistema. Para más información ir a Herramientas, Activos Fijos, facturar orden.
Proveeduría En caso de que la empresa use el Módulo de Proveeduría para llevar un registro y control de los suministros adquiridos como Papelería, lápices, etc. es necesario que registr su saldo inicial por cada bodega.  Para más información ir a Herramientas, Proveeduría, Saldos iniciales.
Acreedores Se deben registrar por cada acreedor, cada una de las facturas por pagar y otras deudas pendientes con sus fechas de vencimiento.  Para más información ir a Herramientas, Acreedores, Saldos iniciales.
Clientes Se deben registrar por cada cliente, cada una de las facturas por cobrar y otras deudas pendientes con sus fechas de vencimiento.  Para más información ir a Herramientas, Clientes, Saldos iniciales
Productos

Es necesario registrar por cada bodega, las existencias de cada producto y su respectivo costo. El costo será el mismo para todas las bodegas y una vez que se ha ingresado los costos de un producto, aparecerá automáticamente al momento de registrar los saldos para otra bodega.  Para más información ir a Herramientas, Productos, Saldos iniciales

Crear Asiento de Diario Inicial

El paso final en la Implementación de DOBRA es la crearción del primer Asiento de Diario que constituye el Balance de Situación inicial de la Empresa. Después de crear el Asiento de Diario inicial el sistema quedará listo para que los usuarios empiecen a registrar transacciones como ventas, compras, cobros, depósitos, etc.

Si el Balance Inicial no está listo al momento de crear el Asiento de Diario inicial, puede crearse un asiento con dos cuentas cualquiera con los valores del DEBE y HABER iguales. Los usuario pueden empezar a trabajar con trasacciones y cuando en la empresa se haya completado el Balance Inicial, se puede modificar el Asiento de Diario número uno para actualizar el Balance que quedó pendiente.
