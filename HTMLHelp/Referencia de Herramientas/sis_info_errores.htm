Módulo de Sistema
Integración OFFLINE de Sucursales

Explicación

El Proceso de Integración de Sucursales proporciona una alternativa para que dos establecimientos que están físicamente incomunicadas, puedan integrarse y trabajar sobre la misma información o sobre una misma Base de Datos.

La Integración OFFLINE no es lo mismo que la REPLICACIÓN de Bases de Datos; en un proceso de replicación se requiere que exista un medio de comunicación ya sea telefónico, microondas, fibra óptica o cualquier otro medio por lo menos mientras dure el proceso de replicación.  La Integración OFFLINE está orientada a establecimientos que no tienen una línea de comunicación de datos permanente.

En este sentido, es correcto entender que debe haber un medio de transporte de información para integrar las transacciones generadas tanto en el establecimiento 1 como en el establecimiento 2.  Además, el Sistema debe ser capaz de realizar procesos de Exportación y de Importación de transacciones para mantener los dos establecimientos con información actualizada.

En DOBRA® el proceso de Integración OFFLINE sigue la metáfora del establecimiento MATRIZ y Sucursales y establece una política de operación basada cuatro etapas: DISTRIBUCIÓN - ACTUALIZACIÓN - EXPORTACIÓN - INTEGRACIÓN como se muestra en la siguiente figura:

Etapas de la Integración OFFLINE de DOBRA®

Como puede verse en la figura, en el proceso de Integración OFFLINE se identifican dos tipos de establecimiento o puntos de procesamiento la MATRIZ y las Sucursales.  En el Proceso sólo puede existir una Matriz y pueden existir varias sucursales pero la Base de Datos Maestra siempre reside en la MATRIZ.

DISTRIBUCIÓN

La distribución consiste en generar una copia exacta de la Base de Datos de la MATRIZ para ser distribuída en las Sucursales.   Este proceso lo puede realizar usando el mecanismo de compresión de DOBRA® o también usando el método BACKUP de SQLServer®.  Se recomienda este último si va a ser llevado a cabo por el Administrador del Sistema.

Note que la Base de Datos pueder ser muy grande para guardarse en un diskette.  Se recomienda en este caso un dispositivo de más capacidad como unidades ZIP de iomega® o el uso de una computadora portátil con unidad de disco duro.

Este proceso sólo debe ser ejecutado en el Servidor de la MATRIZ.

ACTUALIZACIÓN

El proceso de Actualización consiste en sobre-escribir la Base de Datos de Distribución sobre la Base de Datos de la Sucursal.  Note que este proceso sirve para que la sucursal cuente con la última versión de la Base de Datos y si la sucursal ha efectuado transacciones recientemente, y estas transacciones no han sido enviadas a la MATRIZ, entonces se perderán.

La actualización puede ser realizada de dos formas: Usando el Mecanismo de Decompresión de DOBRA® en caso de que la DISTRIBUCIÓN se haya realizado con DOBRA® o usando el Método RESTORE de SQLServer® en caso de que la Distribución se haya realizado usando el método BACKUP de SQLServer®.

Este proceso sólo debe ser ejecutado en el Servidor de la Sucursal.

EXPORTACIÓN

Una vez que la Sucursal ha recibido una actualización de la última versión de la Base de Datos, puede empezar a introducir transacciones.  Las transacciones se guardan con un identificador de Sucursal.   El proceso de Exportación es el mecanismo por el cual una Sucursal exporta sus últimas transacciones a la MATRIZ.  Una Sucursal puede exportar cada día o cada semana y recibir una actualización mensual claro, esto depende de la actividad transaccional tanto de la MATRIZ y de la SUCURSAL.

Para este caso, se usa el mecanismo de compresión de DOBRA® y debido a que sólo se exportan las últimas transacciones, el tamaño de la Base de Datos OFFLINE que se llevará a la MATRIZ podría no acupar más de 1MB que pueden ser copiados perfectamente en un diskette de alta densidad; pero, si el tamaño de la Base de Datos OFFLINE es mayor, se recomienda el uso de unidades de más capacidad como las unidades ZIP de iomega®.

El usuario debe tomar precauciones ya que una vez que se exportan las transacciones nuevas, estas quedan marcadas de tal manera que ya no serán tomadas en cuenta en la próxima exportación; es decir, que si por alguna razón, se dañó o se perdió la Base de Datos OFFLINE, la sucursal quedaría desincronizada de la Base de Datos Central.

Si la Base de Datos OFFLINE recientemente exportada se pierde, la Sucursal debe solicitar una ACTUALIZACIÓN de la Base de Datos de la MATRIZ lo cual invalidará las transacciones nuevas que habían sido exportadas y se tendrán que reingresar en el Sistema.

Para el proceso de exportación se usa un método de compresión y codificación de nombres de tablas y archivos como medida de seguridad para proteger la información que es trasladada desde la Sucursal hasta la MATRIZ.

Este proceso sólo debe ser ejecutado en el Servidor de la Sucursal.

IMPORTACIÓN

El Proceso de Importación e Integración consiste en recuperar las transacciones de una Base de Datos OFFLINE proveniente de una Sucursal e integrarlas a la Base de Datos Central.  Con este proceso se completa el ciclo de Integración OFFLINE.

El usuario debe tener precaución ya que una vez que se han integrado las transacciones provenientes de la sucursal, no se puede repetir el proceso ya que esto generaría duplicaciones de datos y DOBRA® interrumpiría el proceso presentando los errores correspondientes.

Este proceso sólo debe ser ejecutado en el Servidor de la MATRIZ.

Requisitos

1 Lista de Sucursales Una Lista de las Sucursales que participarán en el proceso.  Las Sucursales tienen un código y una descripción.  El código de una sucursal está formado por 2 dígitos siendo el 00 reservado para identificar la MATRIZ.

Para iniciar la ventana de mantenimiento de sucursales seleccione la opción Sucursales de la sección de mantenimiento del módulo de Sistema en el Menú de Inicio de DOBRA®.
2 Instalar código de sucursal en cada computadora. Para realizar este paso, debe seleccionar la sucursal correspondiente desde la ventana de opciones de empresa.

Para iniciar la ventana de Opciones de Empresa seleccione la opción Empresa de la sección de mantenimiento del Módulo de Sistema en el Menú de Inicio de DOBRA®.

Ejecución

Para Iniciar el Asistente de Integración OFFLINE de Sucursales, debe seleccionar la opción Integración de la sección de Asistentes del Módulo de Sistema en el Menú de Inicio de DOBRA®.

Recomendaciones

1 Ejecutar fuera de Horas de Oficina Se recomienda que los procesos sean ejecutados mientras ya nadie está trabajando en el Sistema.
2 Ejecutar en el Servidor Se recomienda que los procesos de Integración sean ejecutados en el Servidor.  DOBRA® genera archivos temporales de las exportaciones e importaciones que pueden ser usados en caso de fallas o errores.  Es conveniente que estos archivos estén en el Servidor.
3 Usar respaldo de SQLServer® para los procesos de Distribución Aunque DOBRA® proporciona un método seguro de para generar la Base de Datos de distribución, este proceso es más lento que el proceso de respaldo de SQLServer®.
